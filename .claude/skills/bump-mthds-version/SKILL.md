# Bump Minimum mthds-agent Version

This skill updates the `min_mthds_version` across all plugin skills in a single coordinated operation.

The canonical source of truth for `min_mthds_version` is `targets/defaults.toml`. All templates receive this value via the Jinja2 variable `{{ min_mthds_version }}` — no manual edits to individual skill files are needed.

## Workflow

### 1. Determine the new version

Ask the user for the target version (semver `X.Y.Z`) unless they already specified it.

Show the current canonical version by reading `targets/defaults.toml` — the `[vars].min_mthds_version` field.

### 2. Update the canonical source

Edit `targets/defaults.toml`: change `min_mthds_version = "OLD"` to `min_mthds_version = "NEW"`.

This is the **only file** that needs a manual edit. All other version references are either:
- Template variables in `.j2` files (`{{ min_mthds_version }}`) — resolved at build time
- Generated build artifacts (`SKILL.md`, `mthds-agent-guide.md`) — regenerated by the build

### 3. Regenerate all build artifacts

Run `make build` from the repo root. This renders all `.j2` templates for all targets (prod + dev + codex) and writes the corresponding output files with the new version baked in.

### 4. Verify with `make check`

Run `make check` from the repo root. This validates:
- All SKILL.md frontmatter `min_mthds_version` values match the canonical version
- All generated files are fresh (match their `.j2` source)
- Target plugin versions are consistent
- Claude and Codex marketplace plugin entries match target configs

If it fails, report the errors and fix them before continuing.

### 5. Report

List all modified files and show the version change: `OLD → NEW`.
